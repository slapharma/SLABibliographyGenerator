Design Overview

7 features across 4 areas. Review each section and confirm before we move to the spec.

Area 1 — Bibliography Enhancements
📝 Paper Annotations

Each row in a bibliography gets an inline "Add note" button. Notes stored in a new note column on bibliography_papers. Shown as grey italic text below the paper title. Editable in-place. Exported in Excel column.

📅 Date Added Field

bibliography_papers.added_at already exists (it's created_at). Surface it in the UI as "Added" column. Add "Date Added" as a sort option and filter range in the bibliography detail filter panel.

🏷️ Description & Tags

Surface the existing description field on bib cards and the detail page (editable inline). Add a tags text column (comma-separated). Tags rendered as coloured chips. Filter bibliographies list by tag.

💬 Inline Abstract Preview

Expand/collapse toggle on every paper row (search results + bibliography). Abstract already exists in the Paper type. No API change needed — pure frontend toggle. Falls back gracefully when abstract is absent.

Area 2 — Export & Sharing
📋 Citation Formatter

Client-side formatter — no API. Formats Vancouver, APA 7, Harvard from the data already in the Paper object. Per-paper "Copy citation" button + "Copy all" for the full bibliography. Format selector dropdown (persisted in localStorage).

🖨️ PDF Export

Opens a print-optimised view at /bibliographies/:id/print — SLA logo, bibliography name, creator, date, numbered reference list in chosen citation format. Uses window.print(). No server-side PDF generation needed.

🔗 Shareable Link

Add share_token (UUID) + is_shared boolean to bibliographies table. New public route /share/:token calls a new /api/share/:token Edge function (no auth). Toggle sharing on/off in the bibliography detail page header.

Area 3 — Saved Search Enhancement
🔁 Re-run + New Result Highlighting

"Run now →" button on saved search cards navigates to /search?savedId=X, pre-fills params, and auto-triggers the search. To highlight new results: store the list of paper IDs from the last run in a new last_result_ids JSON column on saved_searches. On re-run, any paper ID not in that list gets a "NEW" badge. After viewing, the stored IDs update automatically.

Database changes summary
Table New columns
bibliography_papers note TEXT DEFAULT '' — paper annotation
bibliographies tags TEXT DEFAULT '', share_token TEXT, is_shared BOOLEAN DEFAULT false
saved_searches last_result_ids JSONB DEFAULT '[]'

All added with ALTER TABLE … ADD COLUMN IF NOT EXISTS — safe to run on existing data.
